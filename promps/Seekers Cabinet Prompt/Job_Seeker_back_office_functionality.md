# Звіт: Функціональність бек-офісу Шукача (Job Seeker)

## 📋 Загальна концепція

URL базові роути: `/dashboard/seeker` | Middleware: `auth, role:seeker`

Кабінет Шукача розроблений як **дзеркальна система** до бек-офісу роботодавця, але з перспективи пошукача роботи. Вся комунікація та статуси синхронізуються в реальному часі.

---

## 🎯 Основні сторінки та функції

### 1. Дашборд `/dashboard/seeker`

**Призначення:** Швидкий огляд активності та ключових метрик

**Компоненти:**
- **Картки статистики:**
  - 📊 Всього заявок подано
  - 👁️ Активних вакансій у стані розгляду
  - ✅ Запрошень на собесіди (майбутні)
  - ❌ Отримано відмов
  - 💼 Пройдено собесід

- **Недавня активність (Timeline):**
  - Нові запрошення на собесіди
  - Повідомлення від роботодавців
  - Статуси змінилися на "Запрошена на інтерв'ю"
  - Розпочаті тестові завдання

  - Компанія, посада, дата/час

  - Посилання на відеозустріч (Google Meet, Zoom)
  - Кнопка "Нагадування за 1 годину" (опціонально)

- **Рекомендовані вакансії (Smart Recommendations):**
  - На основі профілю та історії вакансій

### 2. Профіль шукача `/dashboard/seeker/profile`

**Призначення:** Управління персональною та професійною інформацією

**Функції:**

#### A. Персональна інформація (Personal Info)
- Фотографія профілю (avatar)
- ПІБ
- Email (можна змінити через верифікацію)
- Телефон
- Місто / Країна (з CitySearch компонента)
- Дата народження (опціонально)

#### B. Професійна інформація (Professional Info)
- Поточна посада (посада і компанія)
- Роки досвіду (розраховується автоматично)
- Сфера експертизи (наприклад: "Backend Development", "Product Design")
- Про себе (біографія, до 500 символів)

#### C. Соціальні мережі та портфоліо
- LinkedIn профіль (посилання)
- GitHub (посилання)
- Портфоліо вебсайт
- Twitter / інші соцмережі (опціонально)

#### D. Пошукові вподобання (Job Search Preferences)
- Бажана посада (multi-select)
- Географія (міста, remote)
- Готовність до переміщення (yes/no/maybe)
- Тип зайнятості (full-time, part-time, contract)
- Бажана зарплата (від/до)
- Мови (та рівень)

#### E. Комунікаційні налаштування
- Лист розсилка (opt-in)
- Email сповіщення (частота: immediately/daily/weekly)
- Приватність профілю (public/private/visible only to employers)

**Функції:**

#### A. Завантаження резюме
- Завантаження файлу (PDF, DOCX, DOC)
- Автоматичне парсування (текст, контакти, навички)
- Перегляд у вбудованому viewer
- Встановлення як основне

#### B. Конструктор резюме (опціонально)
- Шаблони резюме
- Редактор у вікні браузера
- Експорт в PDF/DOCX

#### C. Версіонування
- Історія завантажених резюме з датами
- Можливість обрати резюме для кожної вакансії
- Видалення старих версій

#### D. Рекомендації щодо якості
- Оцінка якості резюме (%)
- Підказки для поліпшення
- Рівень завершеності профілю

### 4. Вакансії `/dashboard/seeker/vacancies`

**Призначення:** Пошук, перегляд та управління вакансіями

**Функції:**

#### A. Пошук і фільтри
- Пошук за текстом (посада, компанія, навички)
- Фільтри:
  - По типу зайнятості
  - По місцю розташування
  - По зарплаті (від/до)
  - По компанії
  - По датам (останні 7, 30 днів)
  - По статусу (не подавав / подав / запрошена на собесіду / offer)

#### B. Вид списку вакансій
- Карточка з посадою, компанією, місцем, зарплатою, типом
- Кнопки швидких дій:
  - ❤️ Додати в улюблені
  - 📝 Подати заявку
  - 👁️ Переглянути детально

#### C. Детальний перегляд вакансії
- Повна інформація про вакансію
- Інформація про компанію (рейтинг, кількість місцевих файлів)
- Розділи:
  - Опис посади та вимоги
  - Рівень експертизи
  - Інформація про компанію
  - Переваги
  - Оклад

#### D. Збережені вакансії (Saved Jobs)
- Організація у папки
- Синхронізація з сервером
- Сортування за датою, релевантністю

**Дизайн карточки вакансії:**

### 5. Мої заявки `/dashboard/seeker/applications`

**Призначення:** Відслідкування статусів всіх поданих заявок

**Функції:**

#### A. Список заявок з фільтрами
- Фільтрація за статусом:
  - 📨 Подана (submitted)
  - 👀 Переглянута роботодавцем (viewed)
  - 🤔 На розгляді (screening)
  - 🧪 Тестування (testing)
  - 📞 Запрошена на собесіду (interview)
  - 💌 Пропозиція (offer)
  - ❌ Відмова (rejected)
  - ⏸️ На паузі (on hold)

#### B. Метрики та статистика
- Всього заявок
- По кожному статусу (counter)
- Успішний коефіцієнт (приблизно: пропозиції / заявки)
- Середній час на рішення

#### C. Сортування
- По даті подачі
- По компанії
- По статусу
- По останній активності

#### D. Пагінація
- 20 / 50 вакансій на сторінці

### 6. Деталь заявки `/dashboard/seeker/applications/{applicationId}`

**Призначення:** Повна інформація про конкретну заявку та комунікацію

**Функції:**

#### A. Інформація про заявку
- Посада та компанія
- Дата подачі
- Поточний статус з історією змін
- Резюме, яке було надіслано
- Опис вакансії

#### B. Статус та переходи
- Поточний статус (큰bold)
- Доступні дії (переживаній на наступний статус)
- Лінія часу всіх переходів статусу з датами

#### C. Тестові завдання (якщо є)
- Назва тесту
- Посилання / файл
- Дата початку
- Дата закінчення
- Статус (не розпочато / в процесі / завершено / знецінено)
- [Завантажити результати]

#### D. Собесіди (якщо є)
- Тип собесіди (technical, HR, final, etc.)
- Дата, час, тривалість
- Формат (online/offline/phone)
- Посилання на відеозустріч (Google Meet, Zoom)
- Адреса офісу (якщо очна)
- Рекомендації від роботодавця (якщо є)
- [Підтвердити] [Я не можу прийти]

#### E. Комунікація
- Чат з роботодавцем (якщо вони інініціюють)
- Вбудований чат або посилання на чат
- Історія повідомлень від роботодавця
- [Відповісти] / [Написати повідомлення]

#### F. Нотатки та оцінки (від роботодавця)
- Оцінка кандидата (якщо видима)
- Нотатки від роботодавця (якщо видимі)
- Рівень цікавості (hot / warm / cold)

#### G. Дії
- [Відкликати заявку] (якщо статус дозволяє)
- [Написати запитання] (звернутися до компанії)

### 7. Собесіди `/dashboard/seeker/interviews`

**Призначення:** Управління запланованими собесідами та підготовка

**Функції:**

## 📊 Синхронізація в базі даних

**Тригери для реальної синхронізації:**

```sql
-- Коли роботодавець змінює статус заявки
CREATE TRIGGER on_application_status_change
AFTER UPDATE OF status ON applications
FOR EACH ROW
EXECUTE FUNCTION notify_seeker_of_status_change();

-- Коли планується собесіда
CREATE TRIGGER on_interview_scheduled
AFTER INSERT ON interviews
FOR EACH ROW
EXECUTE FUNCTION notify_seeker_of_interview();

-- Коли пропонується оффер
CREATE TRIGGER on_offer_created
AFTER INSERT ON offers
FOR EACH ROW
EXECUTE FUNCTION notify_seeker_of_offer();

-- Коли роботодавець відправляє повідомлення
CREATE TRIGGER on_message_sent
AFTER INSERT ON messages
FOR EACH ROW
EXECUTE FUNCTION notify_seeker_of_message();
```
