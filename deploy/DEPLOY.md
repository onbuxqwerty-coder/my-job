# Production Deployment Guide

## 1. First-Time Server Setup

```bash
# Clone & install
git clone https://github.com/you/my-job.git /var/www/my-job
cd /var/www/my-job

composer install --optimize-autoloader --no-dev
npm ci && npm run build

cp .env.example .env
php artisan key:generate

# Storage symlink (so uploaded resumes/logos are web-accessible)
php artisan storage:link

# Database
php artisan migrate --force

# Warm caches
php artisan optimize          # caches config + routes + views
```

## 2. .env Keys You MUST Set Before Going Live

| Key | Where to get it |
|-----|-----------------|
| `APP_KEY` | Generated by `php artisan key:generate` |
| `APP_URL` | Your production domain, e.g. `https://myjob.com` |
| `DB_PASSWORD` | MySQL user password |
| `TELEGRAM_TOKEN` | @BotFather → /newbot |
| `TELEGRAM_WEBHOOK_URL` | `https://yourdomain.com/telegram/webhook` |
| `TELEGRAM_ERROR_CHAT_ID` | Send `/start` to @userinfobot to get your chat ID |
| `STRIPE_KEY` | Stripe Dashboard → Developers → API keys → Publishable |
| `STRIPE_SECRET` | Stripe Dashboard → Developers → API keys → Secret |
| `STRIPE_WEBHOOK_SECRET` | Stripe Dashboard → Webhooks → your endpoint → Signing secret |

## 3. Register Telegram Webhook

Run once after deploy (or after domain change):

```bash
php artisan nutgram:hook:set
# Or manually:
curl -X POST "https://api.telegram.org/bot${TELEGRAM_TOKEN}/setWebhook" \
     -d "url=${TELEGRAM_WEBHOOK_URL}"
```

## 4. Register Stripe Webhook

In Stripe Dashboard → Developers → Webhooks:
- Add endpoint: `https://yourdomain.com/stripe/webhook`
- Select event: `checkout.session.completed`
- Copy the **Signing secret** → paste into `STRIPE_WEBHOOK_SECRET` in `.env`

## 5. Queue Workers & Scheduler (Supervisor)

```bash
# Copy config
sudo cp /var/www/my-job/deploy/supervisor.conf /etc/supervisor/conf.d/my-job.conf

# Adjust /var/www/my-job path in the file if needed, then:
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start my-job-worker:*
sudo supervisorctl start my-job-scheduler
```

Check status:
```bash
sudo supervisorctl status
```

## 6. Scheduled Tasks (handled by Supervisor — no cron needed)

The `my-job-scheduler` Supervisor program runs `schedule:run` every 60 seconds.
If you prefer a system cron instead, add to root crontab:
```
* * * * * www-data php /var/www/my-job/artisan schedule:run >> /dev/null 2>&1
```

### What the scheduler runs automatically

| Schedule | Command | Purpose |
|----------|---------|---------|
| Every hour | `app:send-vacancy-alerts` | Sends new vacancy alerts to Telegram subscribers |
| Daily | `app:deactivate-expired-featured` | Removes featured status from expired promotions |
| Daily | `app:cleanup-temp-uploads --hours=24` | Deletes orphaned Livewire temp files |

## 7. Deploying Updates

```bash
cd /var/www/my-job
git pull origin main
composer install --optimize-autoloader --no-dev
npm ci && npm run build
php artisan migrate --force
php artisan optimize
sudo supervisorctl restart my-job-worker:*
```

## 8. Health Check

```
GET https://yourdomain.com/health
```

Returns `200 {"status":"ok","db":"ok","cache":"ok"}` when everything is running.
Returns `503` if DB or cache is down — use this for uptime monitors (UptimeRobot, etc.).

## 9. Error Monitoring

**Option A — Built-in Telegram alerts (already configured)**
- Set `TELEGRAM_ERROR_CHAT_ID` to your personal Telegram chat ID.
- Any 500 error in production sends a message with the error, file, and URL.

**Option B — Sentry (recommended for teams)**
```bash
composer require sentry/sentry-laravel
php artisan sentry:publish --dsn=https://...@sentry.io/...
```
Then add `SENTRY_LARAVEL_DSN` to `.env`.
